# Plano: Correção de Multi-Temporadas (Season Packs) — v2

**Problema:** 29 torrents são packs multi-temporada (ex: "Season 1 2 3 4 5 6 7 8"), mas o sistema trata como temporada única. Regex puro é frágil — precisamos de uma abordagem semântica.

**Princípio:** O TMDB já sabe quantas temporadas cada série tem. Usar isso como âncora, não regex.

---

## Estratégia de classificação (3 camadas)

### Camada 1 — TMDB-informed (principal, cobre ~80%)
**Pré-condição:** Série tem `tmdb_id`.

**Lógica:**
1. Buscar `season_count` do TMDB (já temos via `getTV()`)
2. Extrair **todos** os números do título do torrent que aparecem perto de "Season"/"Temporada"
3. Comparar com o total de temporadas da série:
   - Se os números cobrem **todas** as temporadas → `coverage = full` (1 a N)
   - Se cobre um subconjunto → `coverage = partial` (ex: 1-3 de 8)
   - Se é temporada única → `coverage = single` (ex: S05)
4. Se não há números claros mas o título contém "Complete" / "Completa" → `coverage = full`

**Exemplo:**
- Torrent: "The Vampire Diaries Season 1 2 3 4 5 6 7 8"
- TMDB: 8 temporadas
- Números extraídos: [1,2,3,4,5,6,7,8] → cobre todas → **full**
- Torrent: "Criminal Minds Season 1-15"
- TMDB: 17 temporadas (série ainda no ar)
- Números: [1,15] → range 1-15 de 17 → **partial (1-15)**

### Camada 2 — LLM (fallback, cobre ~15%)
**Quando usar:** TMDB indisponível OU números extraídos são ambíguos OU sem tmdb_id.

**Prompt (usando Hermes/OpenRouter):**
```
Série: {title} (TMDB: {season_count} temporadas)
Torrent: "{torrent_title}"

Quais temporadas este torrent cobre?
Responda APENAS com JSON: {"seasons": [1,2,3]} ou {"seasons": "full"} ou {"seasons": "unknown"}
```

**Custo:** ~100 tokens por classificação. Para 29 torrents = 2.900 tokens ≈ $0.001 via OpenRouter.

### Camada 3 — Heurística (último recurso, ~5%)
**Quando usar:** LLM indisponível ou timeout.

**Lógica:** Regex básico como fallback mínimo:
- `S0\d` → single season
- `Season 1-8` → range
- Se nada funcionar → `unknown`

## Implementação (4 cards)

### Card 1 — Função `classifySeasonCoverage()`
**Arquivo:** `src/modules/torrent/season-coverage.ts` (NOVO)

```typescript
interface SeasonCoverage {
  type: 'full' | 'partial' | 'single' | 'unknown'
  seasons: number[]       // [1,2,3,4,5,6,7,8] ou [1] ou []
  confidence: 'tmdb' | 'llm' | 'heuristic'
}

async function classifySeasonCoverage(
  torrentTitle: string,
  tmdbId: number | null,
  seasonCount: number | null  // from TMDB
): Promise<SeasonCoverage>
```

**Steps:**
1. Extrair números do título (regex solto: `/\d+/g` perto de "season"/"temporada")
2. Se temos `tmdbId` + `seasonCount`:
   - Comparar números extraídos com seasonCount
   - Se cobrem tudo → `{type:'full', seasons:[1..N], confidence:'tmdb'}`
   - Se cobrem subset → `{type:'partial', seasons:[...], confidence:'tmdb'}`
3. Se sem TMDB ou ambíguo → chamar LLM (Camada 2)
4. Se LLM falhar → heurística (Camada 3)
5. Cachear resultado em `torrent_episodes` ou nova tabela leve

### Card 2 — Atualizar `resolveEpisodes` para usar `SeasonCoverage`
**Arquivo:** `src/modules/torrent/episodes.ts`

**Mudança:** Em vez de usar `content_torrents.season`, chamar `classifySeasonCoverage()`:
- Se `type === 'full'` → gerar episódios para todas as temporadas
- Se `type === 'partial'` → gerar para as temporadas listadas
- Se `type === 'single'` → comportamento atual
- `file_index` estimado sequencialmente

### Card 3 — Atualizar catálogo `/:id` para refletir cobertura real
**Arquivo:** `src/modules/catalog/routes.ts`

**Mudança:** Para séries, chamar `classifySeasonCoverage` para cada torrent vinculado e calcular `season_count` real (não baseado em `content_torrents.season`).

**Resultado:** Diários de um Vampiro mostra "8 temporadas" em vez de "1".

### Card 4 — Pipeline: classificar sob demanda + cache
**Arquivo:** `src/jobs/pipeline.ts`

**Mudança:** Após `mergeByTmdbId()`, rodar `classifySeasonCoverage()` para todos os torrents de séries enriquecidas e cachear resultado. Assim o endpoint de catálogo não precisa fazer chamadas LLM em tempo real.

## Exemplo de fluxo

```
Torrent: "The Vampire Diaries Season 1 2 3 4 5 6 7 8 - threesixtyp"
Série vinculada: Diários de um Vampiro (tmdb:18165, 8 seasons)

1. Extrai números: [1,2,3,4,5,6,7,8]
2. TMDB diz 8 temporadas → números cobrem tudo
3. Resultado: { type: 'full', seasons: [1,2,3,4,5,6,7,8], confidence: 'tmdb' }
4. Episodes endpoint gera S01E01..S01E22, S02E01..S02E22, ... S08E01..S08E16
5. Catalog/:id mostra season_count = 8
```

## Torrent sem padrão claro

```
Torrent: "Friends the Reunion"
Série: Friends (tmdb:1668, 10 seasons)

1. Extrai números: [] (nenhum número perto de "season")
2. TMDB disponível mas sem números → ambíguo
3. LLM consultado: "Friends the Reunion" → {"seasons": "unknown"} (é um especial, não temporada)
4. Resultado: { type: 'unknown', seasons: [], confidence: 'llm' }
5. Catalog mostra como "especial" ou não conta como temporada
```

## Vantagens sobre regex puro

| Regex puro | TMDB + LLM |
|---|---|
| "Seasons 1 and 2 - HBO" → quebra | TMDB confirma 2 temporadas → full |
| "Complete Series" → não detecta | TMDB diz 6 seasons → full |
| "S01-S05" → precisa de regex específico | TMDB confirma range → partial |
| "Temporada 1 a 8" → regex em pt-BR falha | LLM entende português |
| "Star Wars 14 Movie Collection" → falso positivo | TMDB: não é série → ignora |

## Riscos
- **LLM custo:** ~$0.001 por classificação. Para 300 torrents = $0.30. Desprezível.
- **LLM latência:** ~1-2s por chamada. Mitigação: cache + pipeline roda offline (não em requisição HTTP).
- **TMDB inconsistente:** série cancelada pode ter menos temporadas que o pack. Mitigação: LLM como árbitro final.
- **Falsos positivos:** "Star Wars Episode 1 2 3" não é série. Mitigação: só classifica se `content.type === 'series'`.
